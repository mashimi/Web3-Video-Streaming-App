Introduction

Streamlify is a Web3 video streaming app, suitable for movies, videos, reviews and streaming. The app allows users to host their videos on the Blockchain and deal with them using CryptoCurrency. The app holders can gain profits from Streamlify by cutting a precentage of the blockchain transactions running on this app for themseleves. It's built on the Ethereum Blockchain and supports MetaMask wallet.

The app was build within an approach to simplify the end user experience of the Web3 apps. It is running perfectly on all viewports, the end user will not suffer from the usual hassles of the Web3 apps. As a user, no need to go into complicated steps to be a part of your app.

Site Identity

In this section we'll change the app identity from ours to yours, such as logo, brand name, etc... You'll find all configs related to this section at \client\src\helpers\settings.js

• Change Logo file and width:
  • Place your new logo into \public folder
  • Place your new favicon into \public folder, give it the name of favicon and make its extension .ico
  • At \settings.js replace the current logo name with your logo name from logo: property
  • Adjust the logoWidth: property number to fit your new logo
• Change Brand Name and Brand Description
  • At \settings.js replace the current brand name with your brand name from brandName: property
  • Replace the current brand description with your brand description from brandDescription: property
  • These new values will reflects at the meta tags and the page titles
• Change Videos available categories
  • We have listed few categories to be available for the minters to choose from, you can remove or add more categories from genresOptions: array.
  • Follow the same naming convention and array structure while adding new category.

Web3 Configuration

In this section we'll change the some important configuration to make your site work properly depending on the network you will choose, all the Web3 Configs will be at \client\src\helpers\settings.js

• Edit the Network Configs:

These configs related to the network you'll use to deploy your smart contracts

  • Replace the current Network Name with your chosen network name from activeNetworkName: property. Any naming convention will work, no restriction here.
  • Change the current RPC URL with your chosen network RPC URL from rpcUrl: preperty. Here you must know the exact RPC URL of your network, you can find it from here https://chainlist.org
  • Change the current Network ID with your chosen network ID from networkId: preperty. Here you must know the exact Chain ID of your network, you can find it from here https://chainlist.org
  • Change the current Block Explorer URL with your chosen network block explorer from blockExplorerUrls: preperty. This url will redirect the user to track his transactions on the blockchain you've chosen.
  • Change the current Currency with your chosen network native token from currency: preperty. For example: if you're deploying your contracts on Binance Smart Chain you may change it to BNB
  • To save the uploaded assets to your own server, as our server will not serve any storage for others, change the current API URL with your API URL from APIURL: property. You should know how to create a custom private URL using Laravel as we mentioned into Laravel Installation section.

Set your Revenue Split from sold videos

In order to gain profits from the app, we cut you a percentage on each video sale, you can change this percentage from \migration\2_deploy_simple_storage.js. From commission: property you can set your own percetage.

Ex 3: Deploying on cPanel

  • Step 1 — Add the Homepage to your package.json File

    Next, open up our \client folder. Open up your package.json file and add a "homepage" attribute like so:

    The format should be "homepage": "http://yourdomainname.whatever"

    Step 2 — Create the build File

    In your application's root directory, run npm install to install the updated dependencies. Once this has finished, the next command you'll run is npm run build.

    You'll notice this creates a new directory in your project called build. The build folder is essentially a super-compressed version of your program that has everything your browser needs to identify and run your app.

    Step 3 — Connect to cPanel

    Let's head over to your hosting provider (Namecheap, Godaddy, Bluehost etc.). Once you've logged in, navigate to the cPanel manager for your domain. Typically there is a dropdown menu of some kind that says "Manage" which will direct you to cPanel.

    Your cPanel manager should look something like this:

    Navigate into the File Manager. There you'll find a dropdown list of directories. The one we're interested in is public_html. Open that up.

    Add the Build File Contents to public_html

    Navigate to the build file in your app's root directory. Open it up and select all the contents inside the build file. If you upload the entire build file itself, the process will not work.

    Once you've copied all the contents inside the build file, upload them into public_html.

    Step 4 — Create and Upload the .htaccess File

    In order for the routes to work in your React app, you need to add a .htaccess file. In the public_html folder, at the same level as the build file contents, add a new file and name it .htaccess.

    Edit the file and insert the following boilerplate information:

    <IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteBase /
    RewriteRule ^index\.html$ - [L]
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteCond %{REQUEST_FILENAME} !-l
    RewriteRule . /index.html [L]

</IfModule>

    You're Super Done.

Installation For Laravel Part

Prerequisites for your Ubuntu server

Before you set up our app, you should know to to prepare the Nginx Server before installing our app, if you dkn't know how to do this, please follow this tutorial https://www.digitalocean.com/community/tutorials/how-to-deploy-a-laravel-application-with-nginx-on-ubuntu-16-04

How to setup our app on your Nginx Server:

Step 1 — Setting Up the Demo Application

  • Upload the folder \streamlify-backend to a Github repository.

  • First, create a directory within the Nginx web root which will hold the application. Because the demo application is named streamlify-backend , let’s use /var/www/html/streamlify-backend 

    sudo mkdir -p /var/www/html/streamlify-backend

  • Move to the new directory and clone the demo application using Git.

    cd /var/www/html/streamlify-backend
git clone https://github.com/YOUR_USERNAME/streamlify-backend .

  • Git will download all of the files from the demo application repository. You’ll see output that looks like this: 

    Git output
Cloning into '.'...
remote: Counting objects: 263, done.

Receiving objects: 100% (263/263), 92.75 KiB | 0 bytes/s, done.
Resolving deltas: 100% (72/72), done.
Checking connectivity... done.

  • Next, we need to install the project dependencies. Laravel utilizes Composer to handle dependency management, which makes it easy to install the necessary packages in one go. 

    composer install

  • The somewhat lengthy output will show the installation progress for all project dependencies: 

    Composer output
Loading composer repositories with package information
Installing dependencies (including require-dev) from lock file
. . .
Generating autoload files
> php artisan clear-compiled
> php artisan optimize
Generating optimized class loader

  • The app itself is set up, so the next step is configuring the app environment. This involves connecting the application and the database and customizing some settings for production.

Step 2 — Configuring the Application Environment

In this step, we’ll modify some security-related application settings, allow the application to connect to the database, and prepare the database for usage. These are necessary steps for all LEMP-backed Laravel applications, not just the demo application we’re using here.

  • Open the Laravel environment configuration file with nano or your favorite text editor. 

    sudo nano /var/www/html/streamlify-backend/.env

  • You’ll need to make the following changes to the file. Make sure to update the placeholder variables, like password and example.com, with the appropriate values. 

    APP_ENV=production
APP_DEBUG=false
APP_KEY=b809vCwvtawRbsG0BmP1tWgnlXQypSKf
APP_URL=http://example.com

DB_HOST=127.0.0.1
DB_DATABASE=laravel
DB_USERNAME=laraveluser
DB_PASSWORD=password

. . .

  • Save the file and exit.

Let’s go through these changes in more detail. There are two configuration blocks here; the first is for application configuration and the second is for database configuration.

In the application configuration section:

  • The APP_ENV variable denotes the system environment in which the application is run. The default value is local which is used for local development environments. For production deployment, it should be changed to production, as we’ve done here. Changing this variable controls log verbosity, caching settings, and how the errors are displayed (depending on the app). With local settings, it’s set up to ease development and debugging, which is convenient while working on an app but shouldn’t be used in a production setup.

  • The APP_DEBUG variable complements APP_ENV and explicitly enables or disables debugging information and verbose error display. On production setups, this value should be set to false to prevent debugging information from being displayed to the users.

  • The APP_URL variable specifies the IP address or domain name under which the site should be accessible. We used example.com domain name here, but you should replace it with your own domain the website should be accessed with.

The database configuration section is a little more straightforward:

  • DB_DATABASE is the name of the database.
  • DB_USERNAME is the name of the MySQL user that the app should use.
  • DB_PASSWORD is the database password for that user.

Development using Ganach Local Server

To start developing on your local machine, make sure that you finalized all the configuration steps, then follow the following steps

  1. Open your terminal in the root directory and run the following command:

    npm install -g truffle

  2. Install the root dependencies by running the following command

    npm install

  3. In your terminal, navigate to the client directory

    cd client/

  4. Install the React.js Web3 dependencies

    npm install

  5. Then move to the root directory again

    cd ..

  6. Open your Ganache local server, make sure it's running.

  7. Move to \client\src\helpers\settings.js and change the following

    • Change activeNetworkName to Ganache local server

    • Change rpcUrl to http://127.0.0.1:7545

    • Change networkId to 5777

    • Change currency to ETH

    • Change APIURL to your own API

  8. Deploy the smart contracts on the Ganache Server

    turffle migrate --reset

  9. Finally, move to the \client directory start the development server 

    cd client
npm start

Smart Contracts Deployments

To Deploy your smart contracts to a test network or a mainnet, make sure that you've configed your files correctly as we outlined at App Configuration section and then follow these steps:

  1. Make sure you've enough funds in your MetaMask account

  2. Open your terminal at the root folder and run the following command:

    truffle migrate --network NETWROK_NAME --reset

    The NETWORK_NAME refers to the name of the network you'll deploy on, exists at truffle-config.js network object

  3. After the command is finished, a new /contracs folder will be generated inside \client\src directory, these are the ABIs of your contracts. If you've not yet upload React part to your server, go with the React installation again to update your server with these ABIs.
  4. You're done!
